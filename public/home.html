JSON Import Format

The import file must be a .json file containing an object where each key is a date in YYYY-MM-DD format and each value is an array of tasks for that day.

Tasks can be plain strings or objects with a text field:

{ "2026-03-10": [ "Team standup at 9am", "Finish project proposal" ], "2026-03-11": [ { "text": "Doctor appointment at 2pm" }, { "text": "Renew car registration", "done": true } ] }

Duplicate tasks (same text on the same day) are skipped automatically. Any entries with an invalid date key or non-array value are skipped with a warning count.
